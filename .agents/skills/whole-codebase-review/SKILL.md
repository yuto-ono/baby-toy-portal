# 全体見直し

このプロジェクトに対して、リポジトリ全体のレビューを行う。「全体見直し」という依頼は、このスキルを使う明示的な合図として扱う。

## 手順

1. まず適用される `AGENTS.md` を読み、レビュー基準として使う。
2. リポジトリ構造、package scripts、ルート構成、共有ライブラリ、テスト、主要な設定ファイルを確認する。
3. パターンを判断できるだけの代表的な実装ファイルを読む。局所的な例だけで全体傾向を決めつけない。
4. 具体的な観点は `AGENTS.md` に委ね、重複して列挙しない。見つけた問題は、どの基準に反しているかを報告内で示す。
5. 必要に応じて読み取り専用コマンドや検証コマンドを実行する。ユーザーが明示的に許可するまで、コード変更、リファクタリング、Issue 起票、stage、commit は行わない。
6. Issue 化が有用な場合は、Issue tracker を利用できるなら既存 Issue を先に検索し、作成前にユーザーへ確認する。

## 報告

重大度順に findings を先に出す。可能な限りファイルと行番号を添える。

各 finding では次を説明する:

- 何が問題またはリスクか
- このリポジトリの基準に照らしてなぜ重要か
- 最小限で現実的な修正方針

対応すべき問題が見つからない場合は、その旨を明確に伝える。実行できなかったコマンドや、全件精査ではなくサンプリング確認に留まった領域など、残る盲点も短く添える。

要約は短くする。レビュー基準から直接導かれる場合を除き、無関係なリファクタリング提案を混ぜない。
